GSoC 2013 Wrap-up: jQuery.IME extensions for Firefox and Chrome

8th August, 2013

I participated in the Google Summer of Code program working for the Wikimedia Foundation. I had a really great time working with the community in last four months. It was by far my best experience while working on an opensource project.

I have been working on the jQuery.IME project this summer. jQuery.IME is a jQuery based input method editor library supporting more than 155 input methods across more than 50 languages. It allows you to write in more than 50 languages using the default keyboard layout. As of now jQuery.IME is provided from the Wikimedia servers. Hence it is only available in Wikimedia projects such as Wikipedia and MediaWiki. The motive behind this WMF project is to provide language technology tools to a wider audience outside the Wikimedia universe. Porting it completely to the client side, in the form of browser extensions, would enable its usage on any website across the web. It would allow users to type in their native language on any website using the same default keyboard layout. For example, one can do a google search in, say, Hindi language.

Proposed deliverables

  • Porting jQuery.IME to Mozilla Firefox addon
  • Porting jQuery.IME to Google Chrome Extension
  • Implement on demand loading of input methods
  • Implement a mechanism to update the extensions from the upstream project with minimal manual effort
  • Integrate Universal Language Selector to provide users an easy way to select any particular language
  • Add support for contenteditable divs in jQuery.IME
  • Add an on screen keyboard feature in jQuery.IME

Completed features

  • jQuery.IME was successfully ported to Chrome and Firefox extensions. Extensions were reviewed and published in their corresponding extension repositories.
    • Firefox extension - https://addons.mozilla.org/en-US/firefox/addon/wikimedia-input-tools/
    • Chrome extension - https://chrome.google.com/webstore/detail/wikimedia-input-tools/fjnfifedbeeeibikgpggddmfbaeccaoh
  • Implemented on demand loading of input methods: Rather than loading all input methods at once, input method scripts are loaded only when the user selects a particular input method
  • The jQuery.ime and jQuery.uls upstream projects were added as git submodules in the extensions. This will provide a way to synchronize the extensions with the upstream projects in the future by simply updating the respective submodule
  • Universal Language Selector (ULS) has been successfully integrated in the extensions, thus providing the users with an easy way to choose among different languages
  • Support for contenteditable divs in jQuery.IME became one of the top priorities of the Language Engineering Team and was implemented by the team itself in very early phase of GSoC.
  • Implemented a basic working version of on-screen keyboard.
    • Demo http://praveensingh.in/projects/jquery.ime/examples/osk.html

Work left to be done

  • The on-screen keyboard implementation is still very basic. Some amount of work needs to be done before it can be merged in the upstream project.
    • Support for contenteditable divs: The current on-screen keyboard implementation works properly only on input boxes and textareas. Contenteditable div implementation is still very buggy.
    • The on-screen keyboard layout needs to be responsive to the page layout.
    • I need to discuss the overall user interface of the keyboard with Pau Giner (Interaction Designer of the Language Engineering team)
  • The extensions do not work properly in facebook chat window. When the page is scrolled, the imeselector is not visible which makes it impossible to select a language. This needs to be fixed.
